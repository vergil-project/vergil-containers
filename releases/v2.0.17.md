
# Release 2.0.17 (2026-05-28)

## Bug fixes

- **install vergil-tooling in bare-runner build jobs**
Add the vergil-actions setup/vergil composite action to the build-scan-push and publish-base jobs before their Trivy scan steps, so vrg-trivy-scan is available on bare ubuntu-latest runners.

- **upgrade pip in python-support fragment to fix CVE-2026-8643**
The Debian python3-pip package (25.1.1+dfsg-1) is vulnerable to CVE-2026-8643 (path traversal via malicious entry point name). This blocks Trivy scans for Go and Rust prod images. Upgrade pip to >=26.1 before installing yamllint and uv, matching the pattern already used in the Python Dockerfile template.

- **audit trivyignore: remove 182 stale entries and update comments**
Scanned all six dev images (base, python, go, rust, ruby, java) against
Trivy 0.70.0 without the ignore file. Cross-referenced the 89 live CVEs
against the 270 entries in .trivyignore.

Removed 182 entries that Trivy no longer reports (fixed upstream in base
images or dependencies). Updated comments on remaining entries to reflect
current fix availability — several Debian packages (gnutls, openssh, jq,
libcap, krb5, nghttp2, cpython) now have fixes that will land on the next
image rebuild. Consolidated duplicate comment blocks and added audit date.

Reduced from 270 to 88 CVE entries (67% reduction).

- **remove redundant vergil-tooling install steps from bare-runner build jobs**
- **replace language: base with container-suffix: base**
The base container suffix is not a language — use the dedicated container-suffix input instead. Ref #311

- **remove pip from non-Python images, install uv via standalone installer**
