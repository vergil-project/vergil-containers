
# Release 2.1.7 (2026-06-30)

## Bug fixes

- **arm package-cleanup deletion correctly per trigger (#388) (#390)**
The dry-run gate used 'cond && a || b', which hits the GitHub Actions falsy-fall-through pitfall: a manual run with 'Preview only' unchecked (inputs.dry_run == false) fell through to the scheduled branch and stayed in preview, so the on-demand delete path never armed. Rewrite as !((manual && dry_run==false) || (schedule && var=='false')) so each trigger maps to its intended value with no fall-through. Document the on-demand one-off delete path.
