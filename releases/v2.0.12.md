
# Release 2.0.12 (2026-05-28)

## Bug fixes

- **pass app secrets to github-config audit**
The upstream ops-github-config reusable workflow now requires a GitHub App token because the Actions permissions API needs administration:read, which GITHUB_TOKEN cannot provide. Forward APP_CLIENT_ID and APP_PRIVATE_KEY to the github-config job.

- **suppress DL3003 for checksum verification subshells**
The (cd /tmp && sha256sum -c) pattern triggers hadolint DL3003 (Use WORKDIR to switch to a directory). This is a false positive: cd is used inside a subshell for temporary directory context during checksum verification, not for persistent directory changes where WORKDIR applies.

- **suppress linux-libc-dev CVE-2024-2193 and CVE-2026-43503**
Both are container false positives — linux-libc-dev is a kernel header package and containers use the host kernel, not the headers in the image. Unblocks the CD pipeline for go, ruby, and rust images. Ref #263

- **suppress Perl Archive::Tar and Go stdlib CVEs blocking dev image publish**
Add CVE-2026-42496 (CRITICAL, symlink extraction), CVE-2026-9538
(HIGH, memory consumption) for Perl Archive::Tar, and CVE-2026-39823
(HIGH, URL handling regression) for Go stdlib to .trivyignore.

No Debian fix available for the Perl CVEs. Go CVE awaits upstream
rebuilds with Go 1.25.10+ / 1.26.3+. None are exercised by the
dev container workloads.

- **suppress additional Perl and Go stdlib CVEs from updated Trivy DB**
Add CVE-2026-8376 (Perl heap buffer overflow, CRITICAL),
CVE-2026-39825 (Go ReverseProxy query forwarding, HIGH), and
CVE-2026-39826 (Go html/template script injection, HIGH).

Trivy DB updated between the first suppression merge and the CD
run, surfacing these additional CVEs. Same rationale: no Debian
fix for Perl, Go binaries are static linters that do not exercise
the affected code paths.

- **switch CI container refs from prod-base to dev-base and bump to 2.0.12**
- **suppress 5 new CVEs blocking dev image publish**
