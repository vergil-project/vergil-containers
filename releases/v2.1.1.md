
# Release 2.1.1 (2026-06-23)

## Bug fixes

- **grant actions: read to the security job (#340)**
ci-security.yml@v2.1.3 requests actions: read from its caller
(codeql-action/upload-sarif needs it on private repositories). GitHub
validates the request against the calling job's permissions at startup,
so the security job must grant it before v2.1 is re-promoted to v2.1.3.
The grant is backward-compatible with v2.1.2.

- **pin pip>=26.1.2 to close CVE-2026-8643 (#345)**
The >=26.1 lower bound floated to a still-vulnerable pip (26.1) at build time. CVE-2026-8643 / PYSEC-2026-196 (entry-point path traversal) affects pip <= 26.1.1 and is fixed in 26.1.2. Bump both the base and python image templates to pip>=26.1.2 and document the advisory in the inline comment alongside the existing CVE-2026-3219 note.

- **pin marketplace source ref to v2.1 (#347)**
The vergil-marketplace source block in .claude/settings.json declared a GitHub repo but omitted the ref field, so vrg-github-repo-config audit flagged the local profile NON-COMPLIANT (expected 'ref = v2.1', actual 'ref = None'). Add the ref pin so the local config audit passes, unblocking release.


## Chores

- **remove gh CVE-2026-48501 ignore and triage CVE-2026-46244**
GitHub's apt repo now publishes gh 2.93.0 on both amd64 and arm64
(index, InRelease hashes, and .deb SHA256s all verified consistent),
so rebuilds pick up the fixed gh CLI and the temporary CVE-2026-48501
suppression is no longer needed. Ref #326.

Also add CVE-2026-46244 (linux-libc-dev, netfilter nft_inner, no
Debian fix) to the kernel false-positive block — it has been failing
the nightly Trivy gate for the go/ruby/rust images since 2026-06-03.
Containers use the host kernel, so kernel-header CVEs are false
positives, consistent with the existing section rationale.

- **triage CVE-2026-42504 to unblock patched-Go image rebuilds**
CVE-2026-42504 (GO-2026-5038, quadratic CPU use in
mime.WordDecoder.DecodeHeader, fixed in go1.25.11 / go1.26.4) entered
the vulnerability DB on 2026-06-05 and now fails the Trivy publish gate
for every image: it flags the prebuilt gh, actionlint, shfmt, scorecard,
and trivy binaries, which are compiled upstream with older Go. None of
these tools decode untrusted MIME headers.

The gate failure blocks the rebuild that delivers the patched Go
toolchain: the go images build FROM the moving golang:1.25 / golang:1.26
tags, which have pointed at go1.25.11 / go1.26.4 upstream since
2026-06-03, so any promoted rebuild ships the fixed stdlib (GO-2026-5037
/ GO-2026-5039) and consumer govulncheck goes green.


## Features

- **add ansible-lint to the common tooling layer (#343)**
Install ansible-lint (pinned 26.4.0) in every dev image so the forthcoming ansible-lint check in vrg-validate (vergil-tooling#1667) has a binary to run. Mirrors yamllint's footprint: pip in the Python-based images (base, python) and uv tool install via the python-support fragment for the rest (go, java, ruby, rust).

ansible-core comes along as a resolved dependency; ansible-lint imports it as a library and lints end-to-end without the ansible-core console scripts on PATH (verified online and offline). Only ansible-lint is pinned, consistent with how yamllint pins itself. Docs and the CLAUDE.md common-layer inventory updated to list the tool.
