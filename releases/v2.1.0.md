## Documentation

- **add design spec for no-cache nightly rebuild (#304)**
Bounded staleness strategy: nightly rebuilds bypass buildx registry cache to guarantee fresh apt packages; intraday CD builds keep full caching for speed.

- **add permissions note to nightly rebuild design (#304)**
- **add implementation plan for no-cache nightly rebuild (#304)**
- **design for OSS container runtime in local build.sh (#322)**
Prefer nerdctl (Lima + containerd) over docker in docker/build.sh, with docker as an optional fallback and a VRG_CONTAINER_RUNTIME override. Records the ecosystem-wide local-prefers-OSS-runtime, CI-stays-on-docker convention and resolves the nerdctl builder-prune divergence (no --filter) by making the prune step runtime-aware. Scoped to vergil-docker; MQ and scripts/dev cleanup tracked separately. Successor to #71 (Colima research, now superseded).

- **move OSS runtime design to docs/specs (#322)**
Relocate the design doc from docs/superpowers/specs to docs/specs to match this repo's established spec location and naming convention.

- **incorporate pushback fixes into OSS runtime design (#322)**
Address four issues from spec pushback review: (1) add a buildkit precondition check in build.sh with a remediation message plus a vergil-vm handoff for guaranteeing buildkitd; (2) validate the resolved runtime so a typo'd or uninstalled VRG_CONTAINER_RUNTIME fails loudly instead of as a raw command-not-found; (3) extend validation to a build->run round-trip confirming the nerdctl-built image is consumable (namespace continuity), not just produced; (4) update the build.sh header comment to describe the runtime-aware prune behaviour.

- **implementation plan for OSS runtime in build.sh (#322)**
Task-by-task plan: make docker/build.sh runtime-agnostic (detection, override + buildkit validation, runtime-aware prune, header comment), soften CLAUDE.md/README docs, and a nerdctl build->run round-trip acceptance check. Derived from the approved design and pushback fixes.

- **align plan with spec: buildkit failure test + vergil-vm handoff task (#322)**
Alignment-check fixes: add a non-invasive buildkit-precondition failure-path test (stubbed nerdctl) to verify the remediation message ships working, and add Task 4 to file the vergil-vm buildkitd tracking issue and cross-reference it from the spec, closing the handoff loop. Spec handoff/risks notes updated to point at the tracking issue.

- **describe OSS runtime preference for local builds (#322)**
Soften the 'Docker is the only prerequisite' design principle to 'a single OCI runtime (nerdctl/Lima preferred, Docker supported)', and note that build.sh auto-detects the runtime (nerdctl, falling back to docker) in the CLAUDE.md and README build sections.

- **link vergil-vm buildkitd tracking issue (#322)**
Record the concrete vergil-vm issue (vergil-project/vergil-vm#97) that owns guaranteeing buildkitd in the Lima VM, closing the handoff loop from the buildkit precondition.
