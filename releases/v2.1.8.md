
# Release 2.1.8 (2026-07-01)

## Bug fixes

- **gate Trivy on actionable CVEs only via trivy.yaml (#395) (#396)**
The docker-publish Trivy gate failed almost daily on unfixable false positives (linux-libc-dev kernel headers, unfixed Debian libs), stranding nightly publishes and releases and forcing per-CVE .trivyignore triage. Add a repo-root trivy.yaml (auto-loaded by both the shared amd64 action and the raw arm64 scan, since both run from /workspace) that sets ignore-unfixed: true and an ignore-policy dropping all linux-libc-dev findings. The gate now fires only on fixed, un-suppressed CVEs (actionable). Verified in-container: config auto-loads, rego compiles and filters by PkgName, and a dev-base rootfs that fails today on 7 unfixed CVEs drops to 0. .trivyignore is retained for fixed-CVE suppressions.
