
# Release 2.1.4 (2026-06-27)

## CI

- **add build-only docs verification job to ci.yml (#369)**
Add a docs job calling ci-docs.yml@v2.0 so docs-build failures are caught pre-merge instead of post-merge in CD. The job id is exactly 'docs' to produce the 'docs / docs' check context required by branch protection.

- **honor native no-cache and unconditionally repopulate buildx cache (#370)**
The cache-from/cache-to expressions used the GitHub Actions fake-ternary
pattern (`inputs.no-cache && '' || format(...)`). With no-cache true,
`true && ''` is falsy, so `||` fell through to the cache ref anyway — the
branch meant to disable caching was unreachable. Nightly "fresh" rebuilds
silently built from the registry cache and never picked up patched
packages.

Pass no-cache through to docker/build-push-action's native input so the
nightly genuinely skips the cache, and make cache-to unconditional
(mode=max) so the fresh rebuild repopulates the cache — healing
subsequent push-triggered CD builds that legitimately use it. Applied to
both the matrix and base publish jobs.
